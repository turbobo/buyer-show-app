## 买家说 — 技术方案文档

### 一、项目概述

「买家说」是一个购物种草社区 H5 应用，用户可以发布购物体验、浏览好物推荐、互动评论点赞。面向移动端优先设计，部署到 EdgeOne Pages。

### 二、技术栈

| 层面 | 选型 | 版本 | 说明 |
|------|------|------|------|
| 框架 | Next.js (App Router) | 14.x | React 元框架，支持 SSG/SSR |
| 语言 | TypeScript | 5.4 | 类型安全 |
| 样式 | Tailwind CSS | 3.4 | 原子化 CSS |
| 动画 | Framer Motion | 11.x | 页面转场、微交互 |
| 状态 | Zustand | 4.5 | 轻量状态管理 (<1KB) |
| 后端 | Supabase | - | PostgreSQL + Auth + Storage（待接入） |
| 部署 | EdgeOne Pages | - | 静态导出，腾讯 CDN |

### 五、UI 设计规范

**色彩系统**：
- 主色：珊瑚红 `#FF6B35`（coral-500）
- 背景：暖白 `#FAF7F5`（warm-50）
- 强调：渐变 `from-coral-500 to-coral-400`

**交互规范**：
- 卡片：圆角 16px + 微阴影 + 按压缩放 (scale 0.98)
- 按钮：圆角胶囊 + 渐变色 + 按压反馈
- 列表：错落淡入动画 (staggerChildren, 60ms 间隔)
- 页面转场：Framer Motion fadeInUp
- 点赞：心跳动画 (scale + rotate 组合)
- 加载：骨架屏 shimmer 动画

**移动端**：
- 最大宽度 430px 居中
- 底部安全区 (safe-area-inset-bottom)
- 触摸反馈 (active:scale)

### 六、数据库设计（Supabase）

```sql
-- 用户资料
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  nickname TEXT NOT NULL,
  avatar_url TEXT DEFAULT '',
  bio TEXT DEFAULT '',
  post_count INT DEFAULT 0,
  follower_count INT DEFAULT 0,
  following_count INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 帖子
CREATE TABLE posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  images TEXT[] DEFAULT '{}',
  tags TEXT[] DEFAULT '{}',
  product_name TEXT DEFAULT '',
  price TEXT DEFAULT '',
  rating INT DEFAULT 5,
  like_count INT DEFAULT 0,
  comment_count INT DEFAULT 0,
  favorite_count INT DEFAULT 0,
  status TEXT DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 评论
CREATE TABLE comments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id UUID REFERENCES posts(id) ON DELETE CASCADE,
  user_id UUID REFERENCES profiles(id),
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 点赞
CREATE TABLE likes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id UUID REFERENCES posts(id) ON DELETE CASCADE,
  user_id UUID REFERENCES profiles(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(post_id, user_id)
);

-- 收藏
CREATE TABLE favorites (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  post_id UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(user_id, post_id)
);
```

### 八、搜索匹配规则

`services/post.ts#searchPosts` 当前匹配策略：

| 字段 | 操作符 | 匹配方式 |
|------|--------|----------|
| `title` | `ilike` | 大小写不敏感模糊包含（`%kw%`） |
| `content` | `ilike` | 大小写不敏感模糊包含 |
| `product_name` | `ilike` | 大小写不敏感模糊包含 |
| `tags` (TEXT[]) | `cs` (contains) | 数组**整段**包含，如 `{数码}` |

PostgREST 通过 `.or(...)` 组合多字段，多条件用逗号分隔。关键词中的 `,`、`(`、`)` 会破坏 or 表达式，调用前会先剥除这三个字符。

**为什么 tags 不用 ilike**：`tags` 是 `TEXT[]`，无法直接 ilike；用 `cs` 整段匹配能命中 `["数码","耳机"]` 这类数组里的标签。代价是只能完全匹配单个 tag，不能模糊（如「数」搜不到「数码」），符合产品「点热门标签 = 精确分类筛选」的预期。

**已知限制 / 后续可优化**：

- 大数据量下 `ilike '%kw%'` 走全表扫描，需建 `pg_trgm GIN` 索引，或升级到 PostgreSQL 全文检索（`tsvector` + `to_tsquery`）。
- `tags` 列建议建 GIN 索引（首页标签筛选已建，搜索复用）。
- 关键词转义目前是黑名单（剥除 `,()`），无法防御 `%`、`_` 这类 ilike 通配符；后续若加入用户自定义复杂查询需做白名单转义。

### 九、错误处理契约

错误处理是工程质量的底线，详细规范见 `技术开发规范.md` §6.2。架构层登记如下：

**核心文件**：

| 文件 | 职责 |
|------|------|
| `src/lib/error-codes.ts` | 业务错误码统一表（通用 1xxx / 认证 2xxx / 校验 3xxx / 资源 4xxx / 业务 5xxx） |
| `src/lib/api-error.ts` | 统一异常类 `ApiError`，封装 code / userMessage / cause |

**抛错与捕获形式**：

- `services/` 层只抛 `ApiError` 实例，禁止抛 `string` 或裸 `Error`
- 调用方用 `ApiError.is(err)` 类型守卫识别业务错误，未识别走兜底文案 + `console.error('[scope]', err)`
- `services/` 层禁止操作 UI（弹 Toast、跳转），UI 关注点全部归调用方

**用户提示分级**：

| 错误类型 | 提示形式 |
|---------|---------|
| 表单实时校验 | 内联红字 + 按钮上方 hint |
| 业务可恢复 | Toast 自动消失 |
| 认证失败 | 弹 LoginSheet 或跳 `/login` |
| 阻断性错误 | 整页错误态 + 返回按钮 |
| 全局兜底 | `app/error.tsx` 错误边界 |

**前后端约束镜像**：数据库 CHECK 约束（如 `posts_content_check: char_length(content) >= 10`）必须在 `services/` 层用对应错误码（`CONTENT_TOO_SHORT`）做前置校验，避免裸 PostgREST 错误信息泄漏给用户。
